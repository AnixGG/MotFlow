# Реализация нейросетевого трекера объектов в реальном времени с использованием недорогих серверных GPU

Это проект про multi-object tracking на спортивных видео. В основе пайплайна используются YOLO, BoTSORT и разные варианты GMC для компенсации движения камеры.

Основная мысль проекта: в tracking pipeline важен не просто самый сложный метод motion estimation, а баланс между точностью и скоростью. В текущих экспериментах `RAFT` дал лучшую скорость, а `sparseOptFlow` показал лучшее качество трекинга, поэтому работа по сути исследует компромисс между `accuracy` и `latency` на доступном GPU.

## Что есть в репозитории

- `src/` — основной код пайплайна
- `configs/` — YAML-конфиги запусков
- `scripts/` — скрипты для запуска 
- `experiments/` — результаты запусков
- `presentation/` — материалы для презентации
- `tmp/results/` — временные результаты

## Как запускать

Базовый запуск:

```bash
python3 scripts/run_baseline.py
```

Экспериментальный запуск:

```bash
python3 scripts/run_experiment.py
```

## Что получается на выходе

После запуска обычно сохраняются:

- `tracks/*.txt` — треки в MOT-формате
- `metrics.csv` — итоговые метрики качества
- `timing.csv` — время работы и FPS
- `run.log` — лог запуска
- `env_info.txt` — информация о среде выполнения
- `vis/*.mp4` — видео с визуализацией треков, если включена визуализация


## Про результаты

На текущий момент временные результаты я отдельно сложил в:

```text
tmp/results/gpuT4
```

Там лежат `metrics.csv` и `timing.csv` для нескольких вариантов GMC.
